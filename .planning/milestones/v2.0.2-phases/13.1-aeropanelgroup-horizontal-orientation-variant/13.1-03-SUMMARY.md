## Decisions Made

### Horizontal Title Rendering: BoxWithConstraints + requiredWidth(maxHeight) + rotate(-90f)

The initial Plan 02 approach used `graphicsLayer { rotationZ = -90f }` with `softWrap=false` + `clipToBounds()` on the containing Box. During sign-off (GAP-1) titles were visually cut off or mis-centered because the Text composable measured as if it had a layout width of 36dp (the strip width), not the strip height (the available length for the rotated text).

Two intermediate fix iterations tried `placeRelativeWithLayer` offset approaches. The final approved approach:

```kotlin
BoxWithConstraints(modifier = Modifier.weight(1f).fillMaxWidth().clipToBounds()) {
    Text(
        text = section.title,
        modifier = Modifier.requiredWidth(maxHeight).rotate(-90f),
        softWrap = false,
    )
}
```

`requiredWidth(maxHeight)` tells the Text its layout width equals the strip's height dimension (the full length available after 90° rotation). `rotate(-90f)` then rotates in draw phase. This is the correct approach; `graphicsLayer`-only and `placeRelativeWithLayer` approaches do not give the Text the right measurement constraints.

### GAP-2 Column Distribution Fix

The outer section `Row` had `weight(1f)` applied, causing all columns to compete for free space via Row's weight algorithm instead of the `distributePx` budget. Fix: removed `weight(1f)` from the outer Row; non-last sections use explicit `.width(distributedDp)` from `distributePx`; the last section absorbs float rounding. Column widths correct and stable.
