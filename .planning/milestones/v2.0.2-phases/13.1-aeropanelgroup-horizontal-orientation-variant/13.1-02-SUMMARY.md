## Decisions Made

### Three Orientation Branch Points

The plan specifies exactly three places where horizontal and vertical differ:

| Branch Point | Vertical | Horizontal |
|---|---|---|
| `BoxWithConstraints` axis | `constraints.maxHeight.toFloat()` | `constraints.maxWidth.toFloat()` |
| Container composable | `Column(Modifier.fillMaxSize())` | `Row(Modifier.fillMaxSize())` |
| Section size modifiers | `fillMaxWidth()` + `.height(dp)` | `fillMaxHeight()` + `.width(dp)` |

Everything else (sizePx state, expandedState, lastExpandedFractionState, isDragging, computeAvailablePx, distributePx, onToggle, onDragBetween, animateFloatAsState, caretRotations, lastExpandedIdx) is shared and runs identically.

### Horizontal Header Strip Layout (36dp wide, full height)

Internal arrangement from top to bottom:
1. `Spacer(8.dp)` — top padding (only when `collapsible=true`)
2. `Icon` (18dp) with `graphicsLayer{rotationZ=caretRotation}` — 0° = ► (expanded), 180° = ◄ (collapsed)
3. `Box(weight(1f).fillMaxWidth().clipToBounds())` containing `Text(softWrap=false, graphicsLayer{rotationZ=-90f})` — rotated bottom-to-top title
4. `Icon` (16dp, explicit tint) — `leadingIcon`, optional
5. `Spacer(6.dp)` + `Column { Row { actions() } }` + `Spacer(8.dp)` — `headerActions`, optional

The `clipToBounds()` on the title Box prevents the drawn-but-rotated text from visually leaking into the content area without any measurement impact (confirmed: `graphicsLayer` is draw-phase only, Pitfall 1).

### Content Area Width

For non-last sections: `.width(animatedMainPx.toDp() - headerWidthDp)` — total animated column width minus 36dp header strip.
For the last expanded section: `.weight(1f)` — fills remaining Row space, absorbing float rounding (PNL-PITFALL-11 pattern applied to X axis).

### caretRotations Chevron Mapping

- Horizontal: `if (expanded) 0f else 180f` — `AeroIcons.CaretRight` points ► at 0° (expanded), ◄ at 180° (collapsed)
- Vertical: `if (expanded) 90f else 0f` — unchanged from Phase 13 shipped model
