# Phase 1 — UI Design Contract

> Visual and interaction contract for the Foundation phase.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> **Scope:** Theme system (AeroColorScheme, AeroTypography, CompositionLocals), three glass
> modifiers (glassEffect / glassPanel / glassSurface), and the :showcase skeleton with theme
> switcher and Foundation demo section. No interactive components (buttons, inputs) are
> in scope for Phase 1 — those arrive in Phase 2.

---

## Design System

| Property          | Value                                                        |
|-------------------|--------------------------------------------------------------|
| Tool              | none (Compose Desktop / Skiko — no web design system)        |
| Preset            | not applicable                                               |
| Component library | Compose Multiplatform 1.10.3 (Material3 bundled)            |
| Icon library      | none in Phase 1 (placeholder icons via Unicode only)         |
| Font              | System default (no custom TTF — Compose Desktop system font) |

**Rationale:** This is a Kotlin/Compose Desktop library project. shadcn, Tailwind, and
web-oriented design systems do not apply. The design contract governs Dp/sp tokens,
Color values, and Compose rendering behaviour directly.

## Spacing Scale

Declared values for Compose Desktop (Dp units, multiples of 4):

| Token | Value  | Usage                                         |
|-------|--------|-----------------------------------------------|
| xs    | 4.dp   | Icon-to-label gap, tight inline padding       |
| sm    | 8.dp   | Compact spacing between grouped elements      |
| md    | 16.dp  | Default content padding, column spacing       |
| lg    | 24.dp  | Section padding, vertical gap between blocks  |
| xl    | 32.dp  | Gap between major sections in showcase        |
| 2xl   | 48.dp  | Top/bottom padding of showcase window content |
| 3xl   | 64.dp  | Reserved — not used in Phase 1               |

**Phase 1 specific exceptions:**
- Glass demo boxes in Foundation section: minimum size 120.dp × 80.dp (tactile target, not touch — desktop)
- Theme switcher button row: `Arrangement.spacedBy(8.dp)` horizontal, `Modifier.padding(16.dp)` outer
- Showcase window initial state: `width = 1200.dp, height = 800.dp` (RESEARCH.md Pattern 4)

*Source: RESEARCH.md §Showcase Window + 8-point standard default*

## Typography

Compose Desktop uses `sp` units. All styles sourced from CONTEXT.md AeroTypography decision
and FOUND-09 requirement. System font is used — no custom typeface loading in Phase 1.

| Role       | Size   | Weight        | Line Height | Compose TextStyle field |
|------------|--------|---------------|-------------|-------------------------|
| title      | 18.sp  | Bold (700)    | 1.2         | `AeroTypography.title`  |
| bodyLarge  | 14.sp  | Normal (400)  | 1.5         | `AeroTypography.bodyLarge` |
| bodyMedium | 13.sp  | Normal (400)  | 1.5         | `AeroTypography.bodyMedium` |
| label      | 11.sp  | Bold (700)    | 1.2         | `AeroTypography.label`  |

**Weights declared:** Normal (400) and Bold (700) only — exactly 2 weights per contract rule.

**Note:** `bodySmall` (12.sp) was removed from the scale. It was 1.sp from `bodyMedium` (13.sp)
and visually indistinguishable at desktop resolution. All former `bodySmall` usages are
consolidated to `bodyMedium` (13.sp Normal).

**Application in Phase 1 showcase:**
- Section headings ("Foundation", "Buttons — coming Phase 2..."): `title` (18.sp Bold)
- Demo box captions ("glassEffect", "glassPanel", "glassSurface"): `label` (11.sp Bold)
- Theme switcher button labels: `bodyLarge` (14.sp Normal) via Material3 Button
- Placeholder text "coming Phase 2...": `bodyMedium` (13.sp Normal)

*Source: CONTEXT.md §AeroTypography, REQUIREMENTS.md FOUND-09, RESEARCH.md §AeroTypography*

## Color

This phase defines three complete color schemes. The contract specifies which theme is
the **default** (AeroBlue) and how the 60/30/10 split maps across all three presets.

### AeroBlue (default theme)

| Role            | Value (hex/ARGB)  | Usage                                      |
|-----------------|-------------------|--------------------------------------------|
| Dominant (60%)  | `#FF0D1B2A`       | Window background (`background` token)     |
| Secondary (30%) | `#CC1A3A5C`       | Card/panel surfaces (`surface` token), `panelBackground` (`#CC152A42`) |
| Accent (10%)    | `#FF4FC3F7`       | `primary` — theme switcher active button, `borderSelected`, caption text only |
| Destructive     | `#FFEF5350`       | `error` — destructive confirmations only (Phase 1: no destructive actions) |

Accent reserved for: theme switcher active-state button background, `borderSelected` on
focused/selected elements, no other uses in Phase 1.

### Glass Rendering Contract

All glass modifiers render via a single `drawBehind {}` block. No `.background()` or `.border()`
calls in the modifier chain (overdraw rule from FOUND-07 and RESEARCH.md Pitfall 2).

| Modifier        | Tokens Used                                | Shadow | Border | Phase 1 Demo Usage   |
|-----------------|--------------------------------------------|--------|--------|----------------------|
| `glassEffect`   | `glassSurface` (gradient), `glassBorder`   | Yes (elevation = 4.dp) | Yes (1.dp) | Demo box #1 |
| `glassPanel`    | `panelBackground` (gradient)               | No     | No     | Demo box #2 — full-width section bg |
| `glassSurface`  | `glassHighlight` (gradient), `glassBorder` | No     | Yes (1.dp) | Demo box #3 |

Default `cornerRadius`: 8.dp for `glassEffect` and `glassSurface`; 0.dp for `glassPanel`.

## Interaction States

Phase 1 delivers no interactive component states (hover, pressed, disabled) — those are
Phase 2. The only interactions in Phase 1 are:

**Primary visual anchor:** the three glass modifier demo boxes (FoundationSection),
differentiated by their glass treatment — each box renders a visually distinct surface
(`glassEffect` with shadow and border, `glassPanel` as full-width background, `glassSurface`
with highlight and border), making them the focal point of the Phase 1 showcase.

| Interaction           | Element              | Behaviour                                                   |
|-----------------------|----------------------|-------------------------------------------------------------|
| Click                 | Theme switcher button | `currentScheme` state updates; full AeroTheme recompose via `staticCompositionLocalOf` |
| Theme switch visual   | All demo boxes       | Immediate color change (no transition animation in Phase 1) |
| Window resize         | Showcase window      | Standard OS-decorated window; Compose layout responds naturally |

No hover animations, focus rings, or pressed states are implemented in Phase 1. The theme
switcher uses plain Material3 `Button` with `containerColor` driven by `colors.primary`
vs `colors.surface` — not AeroButton (that is Phase 2).
