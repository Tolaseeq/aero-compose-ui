<objective>
Eliminate the horizontal-controlled section-duplication bug in `AeroPanelGroup` caused by a write-during-composition violation: the sync loop at `AeroPanelGroup.kt:331-335` writes to `expandedState[i]` in the `AeroPanelGroupImpl` composition body while `expandedState` is read at line 346 in the same `BoxWithConstraints` pass. Inside `BoxWithConstraints` (a `SubcomposeLayout`), this re-subcomposes the children, emitting N extra `key(section.key)` blocks per invalidated pass (3 sections → 9 header strips).

Two surgical changes to `AeroPanelGroupImpl` only:
1. Move the sync loop (lines 331-335) into a `SideEffect { ... }` so it runs after a successful commit, never mid-composition.
2. Replace `val expandedArr = expandedState.toBooleanArray()` (line 346) with `val expandedArr = BooleanArray(sections.size) { i -> isExpanded(sections[i]) }` so size-math reads the authoritative expansion source each composition.

Purpose: Remove the write-during-composition loop without changing the public API, the vertical branch, the animation behavior, or `onLayoutChange` semantics.
Output: Edited `AeroPanelGroup.kt`; the 12 `PanelGroupLogicTest` JVM tests stay GREEN.
</objective>
