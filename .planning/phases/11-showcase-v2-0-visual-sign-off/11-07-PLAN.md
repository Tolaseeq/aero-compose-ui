    Bound column resize so a column cannot be dragged off-screen and the table stays within available
    width. In `AeroDataTable.kt`, in the `widthsDp` `remember(columns, availableWidthPx, resizeOverridesPx.toMap())`
    block, after computing `adjustedDp` and the lower clamp, ALSO clamp the upper bound so the resized
    column cannot exceed the available width minus the sum of the OTHER columns' minimum widths:

    ```kotlin
    val availableDp = availableWidthPx / pxPerDp
    baseWidths.mapIndexed { i, resolved ->
        val overrideDp = (resizeOverridesPx[i] ?: 0f) / pxPerDp
        val adjustedDp = resolved.dpValue + overrideDp
        val minDp = columns[i].minWidth.value
        // Upper bound: available width minus the floor each OTHER column needs, so the table
        // never overflows and no column can be dragged off-screen (F-RESIZE right-side unbounded).
        val othersMinDp = columns.indices
            .filter { it != i }
            .sumOf { columns[it].minWidth.value.toDouble() }
            .toFloat()
        val maxDp = (availableDp - othersMinDp).coerceAtLeast(minDp)
        ResolvedWidth(adjustedDp.coerceIn(minDp, maxDp)).dpValue.dp
    }
    ```

    This keeps the existing lower clamp (no 0dp collapse) and adds the upper clamp (no off-screen growth /
    table stays width-bound). Note: the LEFT over-shrink readability of "Name" is primarily fixed in the
    showcase (11-10) by passing a larger `minWidth` on the Name column; this task guarantees the LIBRARY
    honors whatever minWidth the caller sets as both floor AND part of the others-floor in the max calc.
