# Phase 13.1: AeroPanelGroup horizontal orientation variant - Context

**Gathered:** 2026-06-23
**Status:** Ready for planning

<domain>
## Phase Boundary

Добавить **горизонтальную ориентацию** к существующему `AeroPanelGroup` (требование PNL-HORIZ-01): N секций располагаются **колонками бок-о-бок** вместо строк сверху-вниз. Разделители — **вертикальные**, drag меняет **ширину** соседних раскрытых колонок; свёрнутая колонка ужимается до тонкой **вертикальной полосы-заголовка** (~36dp шириной), соседи забирают освободившуюся ширину. Та же модель VS Code Side Bar, повёрнутая на 90°. Без breaking changes к v2.x API.

Поведенческая семантика (collapse/expand, гибрид controlled/uncontrolled, fraction-state, пер-секционные флаги, drag-clamp, `onLayoutChange` на drag-end+toggle) — **идентична Phase 13**, меняется только ось. Drag-to-reorder, клавиатурный ресайз и вложенность как first-class API остаются вне границы (см. Deferred).

</domain>

<decisions>
## Implementation Decisions

### Публичный API — общее ядро + 2 обёртки (НОВОЕ для 13.1)
- Вынести **общий internal-impl**, принимающий `orientation: Orientation` (вертикаль/горизонталь), и оставить **два публичных входа**:
  - `AeroPanelGroup(...)` — публичная подпись неизменна, дефолт вертикальный (zero breaking change к отгруженному в Phase 13).
  - Тонкая публичная обёртка под горизонталь (имя — на усмотрение Claude, напр. `AeroPanelGroupRow` или `AeroPanelGroup(orientation = …)`; конкретную форму выбрать при планировании, оставаясь аддитивной).
- Один источник лейаут-логики, один scope-DSL `AeroPanelGroupScope` (re-use без дублирования). DSL и пер-секционные параметры (`key`, `title`, `minSize`, `collapsible`, `resizable`, `defaultSize`, `leadingIcon`, `headerActions`) — общие для обеих ориентаций.
- **Следствие (важно для планировщика):** это **рефактор уже отгруженного кода Phase 13**, а не чистое добавление. Существующий `AeroPanelGroup.kt` разбирается на ядро + обёртку. Регрессия вертикали реальна → см. Тесты/Sign-off.

### Расположение заголовка в горизонтали — вертикальная полоса у ведущего края (НОВОЕ)
- Заголовок — **полоса на всю ВЫСОТУ колонки** у ведущего (левого) края, шириной ~36dp; контент — справа от полосы. Это **чистый поворот вертикальной модели на 90°**.
- Свёрнутая колонка = **только эта полоса** (как в вертикали свёрнутая строка = только горизонтальная полоса-заголовок).
- Та же математика `availableForExpanded = totalWidth − Σ заголовки(width) − Σ разделители`; константа «header» теперь интерпретируется как **ширина** полосы, а не высота. Заголовок выглядит одинаково в раскрытом и свёрнутом состоянии.

### Вид свёрнутой/раскрытой полосы — повёрнутый title + шеврон + icon (НОВОЕ)
- В полосе: **title-текст повёрнут на 90°** (IDE tool-window стиль, читается снизу-вверх) через `graphicsLayer { rotationZ = -90f }` (без измерений/доп. layout-проходов), **шеврон сверху**, опциональный `leadingIcon` под ним.
- Идентичность колонки читаема **без опоры на leadingIcon** (текст всегда есть).

### Направление шеврона CaretRight в горизонтали (НОВОЕ)
- **Раскрыт → ► (0°)** — указывает на контент справа. **Свёрнут → ◄ (180°)** — affordance оси раскрытия («нажми, чтобы раскрыть наружу»).
- Анимация поворота — тем же `animateFloatAsState` + `graphicsLayer{rotationZ}` паттерном, что и вертикаль (меняются только целевые углы 0°↔180°).

### Тесты и Sign-off (НОВОЕ — расширенный объём из-за рефактора)
- Pure-logic (`PanelDistribution.kt`) **ориентационно-агностична** — переиспользуется как есть; новые pure-функции не требуются, но существующий `PanelGroupLogicTest.kt` должен остаться **GREEN** после выноса ядра.
- Showcase: **отдельное горизонтальное демо рядом** с существующим вертикальным в `LayoutSection.kt` (append-only; напр. 3 колонки — дерево файлов / редактор / свойства).
- Three-theme sign-off — **на ОБА демо** (вертикальное + горизонтальное) на AeroBlue / AeroDark / Classic. Полный re-sign-off вертикали = страховка от регрессии после рефактора в общее ядро.

### Carried forward from Phase 13 (НЕ переобсуждается — locked)
- **Drag:** `Modifier.aeroDragSplitter(Orientation.Horizontal, …)` (вертикальный грип-разделитель), Pattern 3 + `snap()`-во-время-drag, `rememberUpdatedState(totalWidth)`, прямая запись `sizePx` без анимации.
- **Clamp:** `clampPanelDividerPx` pairwise-модель (minSize непосредственного соседа, не Σ), PITFALL-B `coerceAtLeast`-guard внутри.
- **State:** fraction-as-stable-coordinate, НЕТ `remember(totalPx)`-ключа (PITFALL-A); `key(section.key)` в render-loop; seed-guard `sizePx.size != sections.size`.
- **Раскрытие:** гибрид controlled/uncontrolled по паттерну `AeroAccordion` (`onExpandedChange == null` → uncontrolled), обе ветки намеренные (PNL-08).
- **Divider visibility:** грип только между двумя соседними РАСКРЫТЫМИ секциями (zipWithNext, PNL-06); рядом со свёрнутой — статичный стык.
- **Флаги:** `collapsible=false` (нет шеврона, участвует в ресайзе), `resizable=false` (разделители без грипа, drag off).
- **onLayoutChange:** на drag-end + collapse/expand toggle, НЕ на каждый кадр (PNL-09).
- **Анимация collapse/expand:** 200ms `FastOutSlowInEasing` `animateFloatAsState`.
- **Эстетика:** строго Win7 Aero — `glassPanel` полоса, gloss/gradient/rounded/depth. Контент клипается, скролл — забота вызывающего. Zero new dependencies.

### Claude's Discretion
- Точное публичное имя горизонтальной обёртки и сигнатура (param `orientation` на одной функции vs отдельная функция) — выбрать при планировании, аддитивно и без breaking change.
- Точная внутренняя структура выноса ядра (имена internal-функций/файлов), пока поведение вертикали не меняется.
- Точные dp-отступы/выравнивание повёрнутого текста и иконок в полосе.
- Конкретное содержимое горизонтального showcase-демо.

</decisions>

## Existing Code Insights

### Reusable Assets
- `PanelDistribution.kt` (`distributePx`, `shareTransferOnCollapse/Expand`, `clampPanelDividerPx`, `computeAvailablePx`, `lastExpandedFraction`, `restoreFromFraction`) — **ориентационно-агностичны**, реюз без изменений; «header»/«divider» px трактуются как ширина в горизонтали.
- `Modifier.aeroDragSplitter(orientation, onDrag, onDragEnd, enabled)` — реюз с `Orientation.Horizontal` (курсор E_RESIZE, awaitPointerEventScope без touchSlop).
- `AeroPanelGroupScope` + `section(...)` DSL — общий для обеих ориентаций.
- `glassPanel(cornerRadius=8.dp)` + `AeroIcons.CaretRight` + `graphicsLayer{rotationZ}` — полоса-заголовок и шеврон (углы меняются на 0°↔180°).
- `PanelGroupDivider`-визуал (1dp линия + 3 грип-точки + 8dp hit-area, hover-tint) — поворачивается в вертикальный разделитель.
- `AeroSplitPane` — прецедент компонента с обеими ориентациями в этом же кодбейсе.

### Established Patterns
- **Общее ядро + ориентационный param** — образец `AeroSplitPane` (Orientation внутри одной логики).
- **Повёрнутый текст** — `graphicsLayer { rotationZ = -90f }` без измерений (в тон no-extra-layout-pass правилу проекта).
- **Fraction-as-stable-coordinate**, **live-state в drag-loop** (`rememberUpdatedState`), **hybrid controlled/uncontrolled**, **pure-logic в internal/<feature>/ + JUnit без Compose** — все из Phase 13/Phase 12, переносятся как есть.

### Integration Points
- Рефактор публичного `library/.../components/layout/AeroPanelGroup.kt` → общее ядро + 2 обёртки (вертикальная подпись неизменна).
- Возможный новый internal-файл ядра в `library/.../components/layout/internal/panelgroup/` (на усмотрение Claude).
- Showcase: append-only в `showcase/.../sections/LayoutSection.kt`.
- Без изменений `build.gradle.kts` / `libs.versions.toml` — zero new dependencies.
- **Регрессионная зона:** существующее вертикальное демо + `PanelGroupLogicTest.kt` + Phase 13 success criteria должны остаться рабочими.

## Specific Ideas

- Эталон повёрнутого заголовка — **IDE tool-window** (вертикальная вкладка с текстом снизу-вверх), модель раскрытия — **VS Code Side Bar**, повёрнутая на 90°.
- Шеврон как **affordance оси раскрытия**: раскрыт ► (на контент справа), свёрнут ◄ (раскрыть наружу) — а не state-disclosure-семантика вертикали.
- Эстетика строго Win7 Aero (gloss/gradient/rounded/depth), 200ms-анимации в тон AeroSidebar/AeroAccordion.

</specifics>
